dataProvider#

â€¢ Optional dataProvider: IDataProvider

Optional data provider class instance for API and static resource requests.

description The data provider will override the default MapillaryJS data provider and take responsibility for all IO handling.

The data provider takes precedence over the ViewerOptions.accessToken property.

A data provider instance must implement all members specified in the IDataProvider interface. This can be done by extending the DataProviderBase class or implementing the interface directly.

imageId#

â€¢ Optional imageId: string

Optional image-id to start from. The id can be any Mapillary image. If a id is provided the viewer is bound to that id until it has been fully loaded. If null is provided no image is loaded at viewer initialization and the viewer is not bound to any particular id. Any image can then be navigated to with e.g. viewer.moveTo("<my-image-id>").
